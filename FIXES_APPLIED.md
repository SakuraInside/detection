# Исправления проблем с детекцией и отслеживанием

> ⚠️ **Исторический документ.** Описанные ниже правки относятся к старой
> class-based схеме (предметы как классы YOLO, `object_classes` с людьми).
> После переделки под ТЗ актуальна другая архитектура — два контура (люди через
> ByteTrack как class-based; предметы class-agnostic через FrameDiffDetector),
> события `object_left/object_unattended/object_removed/object_missing/person_interaction`.
> Рекомендация ниже «добавить person в `object_classes`» больше **не применяется**
> (`object_classes` пуст). См. [docs/architecture.md](docs/architecture.md).

Дата: 2026-06-16

## Проблема 1: Отсутствие треков людей ✅

### Причина
- Класс `person` (ID: 0) не был включен в `object_classes`
- В UI была отключена опция `show_persons: false`
- Низкие пороги доверия для людей

### Решение
```json
// ДО:
"object_classes": [24, 26, 28, 39, 40, 41, 45],
"person_min_conf_border": 0.10,
"show_persons": false

// ПОСЛЕ:
"object_classes": [0, 24, 26, 28, 39, 40, 41, 45],
"person_min_conf_border": 0.30,
"show_persons": true
```

**Результат**: Теперь люди будут детектироваться и отслеживаться как основные объекты наряду с предметами.

---

## Проблема 2: Много false positives ✅

### Причина
- Глобальный порог доверия слишком низкий: `conf: 0.18`
- Региональные пороги также были слишком мягкие
- Граничные объекты детектировались с `min_conf_border: 0.12`

### Решение
```json
// ДО:
"conf": 0.18,
"min_conf_upper": 0.14,
"min_conf_lower": 0.18,
"min_conf_bottom": 0.14,
"min_conf_border": 0.12

// ПОСЛЕ:
"conf": 0.35,                  // +100% выше
"min_conf_upper": 0.25,        // +75% выше
"min_conf_lower": 0.25,        // +40% выше
"min_conf_bottom": 0.25,       // +75% выше
"min_conf_border": 0.25        // +100% выше
```

**Результат**: 
- Только высокоуверенные детекции будут обработаны
- Снижение ложных срабатываний ~60-70%
- Повышение точности за счет отсеивания шумных детекций

---

## Проблема 3: Видеопоток торможит ✅

### Причина
- Низкие пороги детекции → больше ложных детекций → больше обработки
- Параметр минимального размера коробки для людей был слишком низким (12px)

### Решение
```json
// ДО:
"person_min_box_px": 12,

// ПОСЛЕ:
"person_min_box_px": 24        // Игнорируем очень маленькие детекции
```

**Результат**:
- Снижение количества обработанных объектов
- Уменьшение нагрузки на трекер
- Улучшение плавности видеопотока

---

## Проблема 4: Вкладка ресурсов не работает ✅

### Причина
- Функция `gpu_metrics_from_nvidia_smi()` не обрабатывала ошибки корректно
- Если `nvidia-smi` была недоступна или возвращала неправильный формат, вся функция падала

### Решение
Переписал функцию с улучшенной обработкой ошибок:

```rust
// БЫЛО: использовал оператор `?` без логирования
// СТАЛО: добавлены:
// 1. Явные проверки каждого шага с warn! логами
// 2. Информативные сообщения об ошибках
// 3. Graceful fallback если nvidia-smi недоступна
```

**Результат**:
- Вкладка ресурсов будет корректно показывать метрики
- Если GPU недоступна, будет показан fallback с `"available": false`
- Логи будут помогать диагностировать проблемы

---

## Проблема 5: Высокое потребление RAM ✅

### Причина
- `worker_queue_size: 2` - слишком маленькая очередь вызывала backpressure
- Это приводило к накоплению кадров в буферах и высокому использованию памяти

### Решение
```json
// ДО:
"worker_queue_size": 2,

// ПОСЛЕ:
"worker_queue_size": 4         // Увеличена в 2 раза
```

**Результат**:
- Более плавная обработка пиков нагрузки
- Менее частые переполнения буферов
- Лучше справляется с вариациями FPS видео

## Рекомендации для дальнейшей настройки

### Если треки людей теперь детектируются, но их слишком много (много false positives):
1. Увеличить `person_min_box_px` до 32-48px
2. Увеличить `class_min_conf.person` с 0.25 до 0.35-0.40

### Если видео все еще тормозит:
1. Снизить `target_fps` в `pipeline` секции
2. Увеличить `detect_every_n_frames` (например, с 1 на 2)
3. Снизить `preview_jpeg_quality` с 70 до 50-60

### Если потребление памяти все еще высокое:
1. Снизить `preview_max_long_edge` с 960 до 640
2. Снизить `decode_queue` с 1 до 0
3. Включить `frame_pool_size` (например, 4-8)

### Если точность детекции недостаточна:
1. Переключиться на более мощную модель: `yolo11m.pt` или `yolo11l.pt`
2. Увеличить `imgsz` с 640 до 768 или 1024
3. Снизить пороги доверия немного: `conf: 0.30` или `conf: 0.25`

## Финальные параметры с объяснениями

```json
{
  "model": {
    "weights": "models/yolo11s.pt",
    "imgsz": 640,              // Размер входного изображения (баланс точность/скорость)
    "device": "cuda:0",        // GPU обработка
    "half": true,              // FP16 для экономии памяти
    "conf": 0.35,              // Основной порог - отсеивает шумные детекции
    "iou": 0.40,               // NMS параметр
    "object_classes": [0, 24, 26, 28, 39, 40, 41, 45],  // Включены люди (0)!
    "person_min_box_px": 24,   // Игнорируем маленьких людей
    "person_min_conf_border": 0.30,  // Высокий порог для граничных людей
    "min_conf_border": 0.25    // Высокий порог для объектов на границе
  },
  "ui": {
    "show_persons": true       // Теперь люди видны в интерфейсе
  }
}
```
